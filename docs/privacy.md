# Privacy Policy

_Last updated: 2026-04-17_

This Privacy Policy explains how **tikbuzz** ("the Software") handles data. The Software is a personal automation tool operated by a single individual (the "Operator") for their own TikTok creator account. It is not offered as a multi-user service and does not collect data from any person other than the Operator.

## 1. Data the Software Handles

The Software handles the following categories of data, all of which belong to the Operator:

1. **TikTok OAuth tokens.** When the Operator authorises the Software, TikTok issues an access token, a refresh token, an `open_id`, the granted scopes, and expiry timestamps. These values are stored locally in a file named `.tiktok-token.json` inside the Operator's working directory.
2. **Video assets generated by the Operator.** The Software produces an MP4 file per run and uploads it through the TikTok Content Posting API. The video file stays on the Operator's local machine; only the uploaded copy leaves the local environment.
3. **Logs produced by the pipeline.** Per-run JSON files (`candidates.json`, `ranked-topic.json`, `brief.json`, `asset-log.json`, `voice-log.json`, `publish-log.json`) are written inside a local `runs/` directory.

No personal data of any third party is collected, stored, or processed.

## 2. How Data Is Used

- OAuth tokens are used only to authenticate requests to the TikTok Content Posting API on behalf of the Operator's own account.
- Logs are used locally for debugging and auditing the pipeline.
- Generated videos are uploaded to the Operator's TikTok account and are not shared with any third party by the Software.

## 3. Data Storage and Security

- All token material and logs are stored on the Operator's local machine.
- `.tiktok-token.json` is excluded from version control.
- The Operator is responsible for the physical and logical security of the machine that runs the Software.

## 4. Data Sharing

The Software does not transmit data to any third party other than:

- **TikTok**, solely to authenticate, upload videos, and query publish status via official APIs.
- **Public web servers** the Operator explicitly configures as trend or asset sources, and only to fetch their publicly available content.

No analytics, telemetry, or advertising SDKs are embedded in the Software.

## 5. Data Retention

- OAuth tokens are retained until the Operator revokes them (via the TikTok app settings) or deletes the local token file.
- Pipeline logs and generated media remain on the local machine until the Operator removes them.

## 6. Rights

Because the Software handles only the Operator's own data, there is no separate data-subject request process. The Operator may delete any local file, revoke the OAuth grant from the TikTok app settings, or remove content from TikTok through TikTok's own interface at any time.

## 7. Children

The Software is not directed to children. Its operator confirms, by using it, that they meet TikTok's own age requirements.

## 8. Changes

This Privacy Policy may be updated from time to time. The most recent version is always the version published on this page.

## 9. Contact

Questions about this Privacy Policy can be directed through the GitHub repository that hosts this page.
